## 二、视觉目标与风格基调

请把整体视觉理解为：

- **不是写实园区**，而是**数字孪生风格的抽象工业园区**。
- **不是卡通风格**，而是**偏商业展示、指挥舱、工业互联网、智慧能源大屏风格**。
- 模型不追求每个螺丝钉的真实细节，而追求：
  - 体块清晰
  - 比例协调
  - 轮廓干净
  - 发光层次明显
  - 结构逻辑合理
  - 与深色科技 UI 完整融合

### 整体视觉应具备的特征

1. **主色调**：
   - 深蓝黑背景
   - 亮青蓝边缘光
   - 亮绿色能量高亮
   - 少量青白色用于高亮文字或扫描线
   - 少量橙色/红色用于告警和异常

2. **质感方向**：
   - 主体为深色金属/深色半透明外壳
   - 轮廓边缘和接缝处有青蓝发光线
   - 能量相关对象带绿色或青绿色脉冲
   - 玻璃、面板、设备边缘具有微弱辉光

3. **空间感方向**：
   - 中心是 3D 园区场景
   - 底部有带科技感的地台和网格
   - 地面存在雷达环、扫描圈、发光路径
   - 场景中允许轻微悬浮、呼吸感、流光扫过

4. **画面构图方向**：
   - 采用**大屏俯视 3/4 视角**，偏等距但不是完全正交
   - 让中央 3D 场景成为绝对视觉中心
   - 左右需要给后续图表和信息面板留空间
   - 整体不应拥挤，模型之间要有明确分区和能量流向

## 四、场景总布局要求

请你构建一个**智慧园区能源管理主视图场景**，建议以一个大型科技底座平台为核心，在平台上布置不同功能区。

### 总体布局建议

将场景组织为以下几个大区块：

1. **园区基础地台 / 科技底座平台**（最底层）
2. **建筑楼块区**（办公楼 / 综合楼 / 数据中心楼）
3. **厂房盒体区**（标准化工业厂房 / 车间）
4. **光伏板阵列区**（整齐排布，规则重复）
5. **储能柜阵列区**（储能集装箱 / 电池柜群）
6. **配电 / 变电 / 能源枢纽区**（变压器、配电节点、塔架或能量交换节点）
7. **园区路网与地面发光路径**
8. **连接各区域的管线 / 电缆通道 / 发光能量路径**
9. **粒子能量流线系统**
10. **标签、指示牌、数据立柱、扫描圈、底部 HUD 光环**

### 坐标和构图建议

请以场景中心 `(0,0,0)` 为基础，构建一个横向展开的园区平台：

- 场景主体占用范围建议在：
  - X 轴：`-60 ~ 60`
  - Z 轴：`-40 ~ 40`
  - Y 轴：`0 ~ 30`

- 平台中心偏中间位置摆放建筑与厂房主体。
- 左前或左下区域放置光伏板阵列。
- 右前或右下区域放置储能柜阵列。
- 后侧或右后区域布置配电节点。
- 各区域之间必须有清晰的能源流向关系：
  - 光伏板阵列 -> 配电节点 / 储能区
  - 储能区 -> 建筑负载 / 厂房区
  - 外部电网 -> 配电节点 -> 园区各区域

## 五、必须实现的模型对象与极细致建模要求

下面这些模型对象必须全部通过代码生成，且每个对象都要统一科技风格。

---

### A. 园区基础地台 / 科技平台（必须实现）

这是整个场景的视觉底盘，决定商业感是否成立。

#### 造型要求

1. 使用一个大尺寸的科技平台作为整个园区底座，不要让模型直接悬空在黑背景中。
2. 底座可由以下几层组成：
   - 最底层：一块大面积深色平台，边角做轻微倒角
   - 中间层：略小一圈的平台，形成层级感
   - 上层：园区实际承载层
3. 平台形状建议为：
   - 圆角矩形
   - 或六边形变体
   - 或带切角的科幻平台
4. 平台边缘必须有发光描边，颜色以青蓝色为主。
5. 平台表面应有以下元素：
   - 发光网格
   - 细微刻线
   - 雷达环/扫描圈
   - 电子电路板风格路径线
   - 局部发光节点

#### 材质要求

- 主体底色：深蓝黑 `#081019` / `#0A1220`
- 边缘发光：`#00D7FF`
- 次级高亮：`#5BFFD1`
- 平台透明度：不需要全透明，但可局部半透明
- 平台表面不要使用写实贴图，尽量使用程序化线条、网格或简单 procedural texture

#### 动效要求

- 底座上存在缓慢扩散的扫描环
- 网格上偶尔有细微流光扫过
- 某些节点缓慢呼吸闪烁

---

### B. 建筑楼块（必须实现）

这是园区中体现“楼宇能耗管理”的主体对象。

#### 建模要求

1. 使用**多个矩形楼块**构成建筑集群，而不是单个盒子草草了事。
2. 每个建筑由以下结构组成：
   - 底部基座
   - 主体楼体
   - 顶部设备层或机房盒体
   - 可选的侧面嵌入式发光条
3. 建筑轮廓要清晰，整体偏现代化楼宇，不要做传统民居风格。
4. 需要至少构建 6 到 12 个楼块，大小和高度有变化，但语言统一。
5. 建筑高度建议分层：
   - 低楼：`4 ~ 8`
   - 中楼：`10 ~ 16`
   - 高楼：`18 ~ 24`
6. 平面尺寸建议：
   - 小楼：`6 x 8`
   - 中楼：`8 x 12`
   - 大楼：`12 x 18`
7. 屋顶不要一片平，建议增加：
   - 小型机房盒体
   - 天线盒 / 空调机组抽象体块
   - 发光边框

#### 细节语言要求

- 立面不需要真实窗户阵列，但必须有“抽象幕墙科技面板”的感觉
- 可以通过以下方式增强楼体精致感：
  - 立面凹槽
  - 竖向灯带
  - 屋顶边缘描线
  - 局部透明外壳
  - 楼体转角加强高光

#### 材质要求

- 主体为深蓝灰材质
- 转角和边缘有亮青蓝描边
- 某些楼体顶部可带浅绿色能耗指示光条
- 被选中时，楼体外轮廓和底座光圈增强

#### 动效要求

- hover 时楼体略微抬升 `0.2 ~ 0.5`
- 被选中时楼体底部出现一圈脉冲环
- 楼体边缘光可做轻微呼吸效果

---

### C. 厂房盒体（必须实现）

这些是工业园区的功能性建筑，应与办公楼区分明显。

#### 建模要求

1. 厂房应比楼块更扁、更长、更稳重。
2. 至少构建 2 到 5 个厂房。
3. 每个厂房建议包含以下层次：
   - 大型长方体主厂房
   - 略微抬起的屋顶或双坡屋顶
   - 正面门区 / 入口框体
   - 侧面通风盒 / 屋顶凸起结构
4. 尺寸建议：
   - 长 `18 ~ 30`
   - 宽 `10 ~ 16`
   - 高 `7 ~ 12`
5. 厂房和楼宇应能一眼区分：
   - 厂房更低矮、更横向
   - 楼宇更竖向、更紧凑

#### 细节要求

- 屋顶可以加入：
  - 发光天窗条
  - 屋脊亮边
  - 局部设备平台
- 正面门区建议做一个略微内凹的矩形框，体现结构层次
- 侧立面可加 2~4 条浅发光横向线条，让其更像工业设施

#### 材质要求

- 色系与楼宇一致，但稍微更偏工业设备风格
- 厂房边角线必须清晰
- 不要用高反光镜面，不要做写实铁皮厂房

#### 动效要求

- 厂房屋顶可存在慢速扫描线
- 厂房进入高能耗/异常状态时，可增强边缘橙红描光

### D. 光伏板阵列（必须实现，重点对象）

这是整个场景最容易出“能源感”和“科技感”的模块之一，必须做精致。

#### 建模要求

1. 通过代码生成**规则整齐的光伏板阵列**。
2. 每个光伏板单元建议由以下部分组成：
   - 薄矩形面板
   - 稍厚一点的边框
   - 支撑腿或支撑架（可以抽象）
3. 光伏板应成组排列，而不是散点摆放。
4. 阵列布局建议：
   - 每组 `4 x 6`、`5 x 8` 或 `6 x 10`
   - 多组阵列并排
5. 每块面板建议有轻微倾角，约 `18° ~ 28°`
6. 阵列所在区域应明显区别于建筑区，通常放在左下或前侧区域。

#### 尺寸建议

- 单块面板：`1.8 x 1.0`
- 厚度：`0.04 ~ 0.08`
- 支架高度：`0.5 ~ 1.0`
- 多组阵列之间保留规律间距

#### 材质要求

- 面板表面底色为深蓝黑玻璃感
- 面板边框带亮青色描边
- 面板内部可用细线模拟电池片分割感，但不要过于真实
- 发电中时，面板表面可出现**横向或斜向流光扫过**
- 能量高时，面板边缘发光增强，并通过 Bloom 形成漂亮的泛光轮廓

#### 动效要求

- 面板表面存在缓慢移动的高亮扫光
- 发电中时，阵列附近有微小粒子向上或向配电节点汇聚
- hover 时，整个阵列轻微增强高亮，不要夸张跳动

#### 性能要求

- 光伏板阵列必须使用**实例化 / 数据驱动复用**，不要手写几百个 mesh

### E. 储能柜阵列（必须实现，重点对象）

储能柜必须带有明显的“设备模块感”和“电池柜感”，要非常适合做数据状态映射。

#### 建模要求

1. 构建多个储能柜/储能箱体，形成有秩序的阵列。
2. 每个储能柜至少由以下结构组成：
   - 底座
   - 主柜体
   - 正面门板分割线
   - 通风栅格/散热槽（抽象化）
   - 侧面能量状态灯带
3. 尺寸建议：
   - 单柜：宽 `1.0 ~ 1.4`
   - 深 `0.8 ~ 1.2`
   - 高 `2.0 ~ 2.8`
4. 一组储能区建议包含：
   - `4 ~ 12` 个储能柜
   - 以 `2 x 4`、`2 x 6`、`3 x 4` 的方式排列
5. 储能区需要设置在园区内比较独立但显眼的位置，通常靠近能源枢纽。

#### 外观要求

- 柜体要有厚重、规整、工业设备感
- 不能只是简单方块，至少要有：
  - 门缝
  - 通风细槽
  - 顶部小设备盒
  - 侧面发光状态条
- 可以在柜体底部加一个发光安装基座

#### 材质要求

- 主体深蓝灰
- 门边和转角用亮青描边
- 侧边的 SOC/功率状态条可以使用绿色或青绿色发光
- 某些高功率状态可让柜体正面局部光更亮

#### 动效要求

- 充电状态：从管线/光伏方向向储能柜汇聚的能量粒子更密集
- 放电状态：从储能柜流向建筑/厂房方向的粒子更密集
- 异常状态：局部红橙色闪烁描边 + 标签告警
- hover 时：储能柜底部出现光圈，柜体轻微亮起

### F. 配电节点 / 变压器 / 能源枢纽（强烈建议实现）

这是连接光伏、储能、楼宇负载的关键中枢，用来让能量流向有“逻辑闭环”。

#### 建模要求

请实现一个或多个能源枢纽对象，可由以下几种抽象体组合：

- 变压器盒体
- 配电柜列
- 支架式能量节点
- 高压塔架的简化版
- 中央能量交换核心台

#### 造型方向

- 不要求真实电力设备建模，但必须表现为“能源路由中心”
- 适合放在场景中后部或中右部
- 需要有更强的发光核心部件，让人一眼知道这是能量分配中心

#### 细节要求

- 核心区域有一个更亮的发光立方体/能量芯
- 周围用框架、支撑杆、线缆连接口增强层次
- 可以加入极简电塔轮廓或配电母线结构

#### 动效要求

- 能源枢纽的发光强度应随系统总负荷轻微呼吸
- 各条能量流线在此汇聚后分流
- 作为主要连接目标，应始终比普通建筑略亮

### G. 园区路网与地面路径（必须实现）

#### 建模要求

1. 需要有简化的道路网络或功能路径，帮助场景具有园区布局感。
2. 道路不要写实柏油路，而要做成科技风地表路径：
   - 深色路径底
   - 两侧细发光边线
   - 局部节点点亮
3. 路网用来组织建筑区、厂房区、光伏区、储能区之间的逻辑关系。

#### 视觉要求

- 可采用浅发光的细线勾勒交通/能量走廊
- 重点路径可加入轻微流动的线性亮带
- 不要抢过主体模型，但要显著提升场景完整度

---

### H. 管线 / 电缆通道 / 能源连接线（必须实现）

这是数字孪生场景最重要的“逻辑纽带”。

#### 建模要求

1. 必须存在一套能量连接关系，而不是各个模块各摆各的。
2. 通过以下几种方式构建：
   - 地表发光路线
   - 半透明管线
   - 悬浮弧形能量通道
   - 地下管线的抽象投影线
3. 核心连接关系至少包括：
   - 光伏板阵列 -> 能源枢纽
   - 光伏板阵列 -> 储能柜
   - 储能柜 -> 建筑楼块
   - 储能柜 -> 厂房盒体
   - 外部电网/塔架 -> 能源枢纽

#### 结构要求

- 使用 `CatmullRomCurve3` 定义路径

### I. 粒子能量流线（必须实现，重点效果）

这是整个数字孪生感最关键的动态语言之一，必须认真做。

#### 功能目标

用粒子 / 发光小球 / 发光精灵，沿着预定义的曲线路径持续运动，表现能量正在流动。

#### 实现要求

1. 每条核心能源路径都应支持粒子流。
2. 粒子系统建议支持以下几种状态：
   - 发电流：光伏 -> 储能 / 枢纽
   - 供电流：储能 -> 楼宇 / 厂房
   - 电网流：外部电网 -> 枢纽 -> 园区
   - 异常流：局部闪烁红橙色
3. 粒子可使用：
   - `Points`
   - billboard sprite
   - 小型发光球体
   - 沿曲线移动的 pulse orb
4. 粒子必须使用加色混合或视觉上接近加色混合的辉光效果。

#### 视觉要求

- 粒子颜色：
  - 电能：亮青蓝 `#00D7FF`
  - 绿色能源：青绿 `#65FFB7`
  - 告警异常：橙红 `#FF7A45` / `#FF4D4F`
- 粒子不要做成烟雾效果，要做成**干净、锐利、精致的发光能量点**
- 能量流线要有快慢层次：
  - 小粒子连续流动
  - 大脉冲球周期性掠过

#### 动效参数建议

- 小粒子速度：`0.4 ~ 1.2` 单位/秒
- 大脉冲球速度：`1.5 ~ 2.5` 单位/秒
- 同一路径可以有 20~120 个粒子，密度根据路径重要性调整
- 粒子之间应有时间错位，不要整排同时移动

#### 数据映射要求

请将粒子流线写成可配置结构，未来可以通过真实数据控制：

- `flowValue` 决定粒子数量和速度
- `flowDirection` 决定方向
- `flowStatus` 决定颜色和是否闪烁

### 材质分层策略

请至少实现以下几类材质：

#### 1. 主体结构材质
用于楼宇、厂房、设备主壳体。

要求：
- 深蓝灰底色
- 金属度低到中等
- 粗糙度中等偏高

要求：
- 深色玻璃感
- 透明度适中
- 带轻微 Fresnel 或边缘提亮感
- 可叠加一层微弱扫光

#### 4. 能量核心材质
用于能源枢纽核心、粒子脉冲球、状态灯带。

要求：
- 更高的 emissive intensity
- 明确进入 Bloom 通道
- 在暗色背景中形成视觉焦点

1. 所有模型都遵循“**深色主体 + 青蓝描边 + 局部荧光绿能量提示**”的统一语言。
2. 发光主要用于：

   - 节点
   - 状态灯
   - 粒子

3. 避免整个模型大面积纯亮色，否则会显得廉价。
4. 以**轮廓发光、接缝发光、能量流动发光**为主，而不是大面积荧光塑料。

## 七、灯光与后处理要求（必须实现）

### 灯光要求

请使用**少量但明确的科技风灯光**，不要搞成写实影视布光。

建议灯光结构：

1. `ambientLight`
   - 强度较低
   - 只用于让阴影区不至于全黑

2. 主方向光 / 主平行光
   - 从偏上方打出整体体积感
   - 光色偏冷白或轻微冷青

3. 边缘补光 / 点光源
   - 使用青蓝色点光源或矩形光感，强化设备边缘
   - 可在能源枢纽、储能区、光伏区附近局部增强

4. 可选雾效
   - 极轻的深色 fog / fogExp2
   - 只用于提升空间层次，不能太浓

### 后处理要求

必须使用 `@react-three/postprocessing` 或等效后处理，实现商业级科技感。

#### 必须包含

1. **Bloom**
   - 重点作用对象：
     - 光伏板边缘
     - 储能柜状态灯带
     - 能源粒子流
     - 扫描圈
     - 发光边框
   - Bloom 不能过爆，必须控制层次
   - 目标是“精致霓虹边缘感”，不是一团白光

2. **抗锯齿**
   - 使用 `SMAA` 或适合的抗锯齿处理
   - 保证边缘线条、细发光线比较干净

3. **可选 Outline**
   - hover 或选中对象时使用轮廓描边
   - 颜色建议青蓝或青白

4. **可选 Noise / Vignette / Scanline（很轻）**
   - 只允许非常轻微地增强氛围
   - 不要做成旧电视滤镜

### Bloom 调性要求

- 亮的应该是“边缘、粒子、扫描环、指示灯、能量核心”
- 不亮的应该是“主体大面积外壳和地面背景”
- 场景整体必须有层次，而不是所有东西一起发光
