# Agent模块设计方案：EcoClaw

> **文档性质**：项目说明书中“Agent 模块”部分的正式说明  
> **文档目标**：说明 EcoClaw 在项目中的定位、核心特征、服务场景与项目价值  
> **适用范围**：智慧园区节能减排综合调度平台中的自然语言调度分析、情景推演、约束注入、应急扩展、设备异常处置、投资规划与历史复盘

## 1. EcoClaw 定位

EcoClaw 不是一个单纯的聊天入口，而是本项目的自然语言调度分析引擎。它位于实时数据链路、优化调度模型和可视化平台之间，负责把用户问题、动态电价、动态碳排、设备状态与运行约束转化为可执行、可展示、可解释的调度结果。

对本项目而言，EcoClaw 最核心的意义不是“接入了一个 Agent”，而是把原本需要人工逐项配置的调度分析过程，转化为一句自然语言即可触发的智能推演流程。用户既可以直接提出 What-if 类问题，也可以使用接近日常业务表达的方式注入约束，系统会自动完成参数解析、约束映射、优化求解、差异归因和建议生成。

因此，EcoClaw 在本项目中承担的是“连接业务问题与优化决策”的角色。它向上连接用户和评委能够直接理解的业务语句，向下连接实时数据、模型求解和页面展示结果，使系统真正形成“感知-决策-展示”的闭环。

## 2. EcoClaw 的三项核心特征

### 2.1 自然语言驱动调度

EcoClaw 的核心能力不是解释已有结果，而是直接驱动调度分析。面对“如果明天电价上涨20%，光伏下降30%，最优方案会怎么变”这类问题，系统会自动识别变量改动、约束对象和影响范围，并将其转化为可执行的情景推演任务。面对“19:00-21:00 电网购电不超过 3000kW”这类业务表达，系统会自动转化为结构化约束并驱动优化求解。

这一特征使 EcoClaw 成为项目中的“自然语言调度入口”。它显著降低了复杂优化模型的使用门槛，使用户不需要理解底层参数和公式，也可以直接发起调度分析。

### 2.2 双信号协同决策

EcoClaw 持续接入并理解园区运行相关的关键数据，包括电价、光照、碳因子和设备状态。它不是在数据更新后被动展示结果，而是围绕“动态电价-动态碳排”双信号协同模型，主动判断外部变化会如何影响成本、碳排、供能结构和设备运行，并快速输出新的综合最优调度方案。

这一特征使 EcoClaw 成为项目中的“碳势感知决策层”可视化入口。它将系统说明书中“进行园区碳排分析预测，并构建动态电价-动态碳排双信号协同模型”的理论目标，落实为可以直接展示和交互的业务能力。

### 2.3 可解释结果落地

EcoClaw 的结果不会停留在一句回答上，而是直接落到 EcoVerse 的工作区中。What-if 推演会形成一句话结论、策略排名变化、关键时段影响、设备调度差异和下一步建议；约束调度会形成约束映射、代价收益对比和调度建议；Pareto 扫描会形成成本-碳排权衡结果；应急、异常和投资类任务则会生成各自对应的工作台内容。

这一特征使 EcoClaw 具备项目中最重要的“展示落地能力”。评委看到的不是抽象对话，而是一套已经形成结构化分析链和可视化结果的完整决策输出。

## 3. EcoClaw 如何服务本项目

EcoClaw 在本项目中的作用主要体现在四类高价值场景，但主场景已经明确转向 What-if 与约束调度。

第一，自然语言 What-if 情景推演。这是 EcoClaw 的绝对核心。用户可以直接描述电价变化、光照变化、购电边界、储氢容量变化或目标权重变化，系统会自动完成推演，并给出最优策略变化、成本/碳排变化、关键时段影响和设备调节建议。这体现了系统围绕“双信号协同调度”的核心价值。

第二，自然语言约束注入。用户可以使用业务语言加入运行约束，例如限制某时段购电上限、要求特定设备优先运行或要求某类能源承担更多负荷，系统会自动完成结构化约束映射并重新求解。这体现了项目不仅能做静态对比，还能响应真实运行管理需求。

第三，扩展型调度任务。面对台风、购电受限、光伏骤降等突发场景，EcoClaw 仍然能够生成应急联动方案；面对燃机温度异常、PEM 压力异常、电解槽电流异常等设备问题，也能够生成异常指挥方案。这些能力保留为重要扩展，但不再作为 EcoClaw 的第一主卖点。

第四，建设规划与复盘分析。面对“光伏扩容后多久回本”这类建设问题，EcoClaw 能生成投资规划结果；面对历史日期和历史工况，也可以结合 What-if 与 Pareto 扫描完成方案论证和复盘分析。

从项目整体来看，EcoClaw 与实时数据链路、优化调度内核和可视化平台是紧密耦合的：它以实时数据为感知基础，以模型求解为决策基础，以工作区和图表页面为结果出口，最终形成面向综合能源调度场景的完整闭环。

## 4. EcoClaw 的项目价值

EcoClaw 的价值，在于它让项目从“展示优化结果的平台”升级为“具备自然语言决策能力的智能调度系统”。它一方面显著降低了复杂调度模型的使用门槛，使用户可以直接用业务语言触发推演和约束求解；另一方面又把实时监测、策略重算、差异解释和结果展示真正串联起来，使系统能够在外部信号变化时快速给出可执行、可展示、可解释的方案。

对评委而言，EcoClaw 最重要的意义不是概念上的 Agent，而是项目核心能力的集中体现。它把“动态电价-动态碳排”双信号协同调度这一论文式、模型式能力，转化为了一个可以直接提问、直接推演、直接看到结论的自然语言调度工作台。这也是 EcoClaw 作为本项目核心亮点的根本价值所在。
